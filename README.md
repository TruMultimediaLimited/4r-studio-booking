# 4R Studio — Booking

স্টুডিও কখন খালি/বুকড তা দেখার একটা পাবলিক পেজ (মাসের ক্যালেন্ডার আকারে), ক্লায়েন্ট নিজে থেকে একটা স্লট বুকিং রিকোয়েস্ট পাঠাতে পারবে, আর টিমের জন্য একটা এডমিন পেজ যেখানে রিকোয়েস্ট কনফার্ম/বাতিল করা যায়।

- পাবলিক পেজ (ক্লায়েন্টদের জন্য): `/`
- টিম পেজ (স্টাফদের জন্য): `/#/admin`

## Local dev

1. `npm install`
2. `.env.example` কপি করে `.env.local` বানান, আপনার Supabase প্রজেক্টের URL ও anon key বসান।
3. `npm run dev`

## Database setup

Supabase Dashboard → SQL Editor-এ গিয়ে `docs/schema.sql` ফাইলের পুরো কনটেন্ট পেস্ট করে "Run" চাপুন। এটা তৈরি করবে:
- `bookings` টেবিল (নাম, ফোন, প্যাকেজ, স্ট্যাটাস — confirmed/pending/cancelled)
- একই সময়ে দুটো বুকিং (বা একটা কনফার্ম্ড + একটা পেন্ডিং রিকোয়েস্ট) ওভারল্যাপ হতে না দেওয়ার ডাটাবেজ-লেভেল গ্যারান্টি
- RLS পলিসি (পাবলিক শুধু ফাঁকা/বুকড দেখতে পারবে ও রিকোয়েস্ট পাঠাতে পারবে, ক্লায়েন্টের নাম/ফোন কখনো পাবলিকলি দেখা যাবে না; শুধু লগইন করা স্টাফ বুকিং কনফার্ম/বাতিল করতে পারবে)
- `public_bookings` ভিউ, যেটা পাবলিক পেজ ব্যবহার করে

এই স্ক্রিপ্টটা যেকোনো সময় আবার রান করা যায় (নতুন ফিচার আসলে, বা টেস্ট ডেটা মুছে ফ্রেশ শুরু করতে চাইলে) — এটা প্রথমে পুরনো টেবিল/ভিউ ডিলিট করে আবার নতুন করে বানায়। **এটা রান করলে তখন পর্যন্ত জমে থাকা সব বুকিং ডেটা মুছে যাবে**, তাই সত্যিকারের বুকিং ডেটা জমার পর অকারণে আর রান করবেন না।

**আগে থেকে চালু ডাটাবেজ থাকলে:** পুরো ফাইল আবার রান করার দরকার নেই — `docs/schema.sql`-এর একদম শেষের নতুন migration ব্লকগুলো কপি করে একবার রান করলেই হবে (এগুলো কোনো ডেটা মোছে না, আগের কোনো কিছুতেও হাত দেয় না):

- **"Security-hardening migration"** — পাবলিক বুকিং রিকোয়েস্টের সার্ভার-সাইড ভ্যালিডেশন (ফোন/নাম/তারিখ/সময়), স্প্যাম-লিমিট, আর সার্ভার-সাইড দাম হিসাব চালু করে। ফ্রন্টএন্ড আর নিজে `total_amount` পাঠায় না — ডাটাবেজই হিসাব করে — তাই **এই ব্লক রান না করা পর্যন্ত নতুন পাবলিক রিকোয়েস্টে দাম/Due রেকর্ড হবে না**।
- **"Portfolio migration"** — পোর্টফোলিও গ্যালারি ফিচারের জন্য `portfolio_items` টেবিল ও `portfolio` স্টোরেজ বাকেট বানায়। **এই ব্লক রান না করা পর্যন্ত এডমিন প্যানেলের "Portfolio Settings" থেকে ছবি আপলোড করা যাবে না** এবং পাবলিক "আমাদের কাজ" পেজে "গ্যালারি লোড করা যায়নি" এরর দেখাবে।
- **"Portfolio albums migration"** — অ্যালবাম/ফোল্ডার সিস্টেমের জন্য `portfolio_albums` টেবিল ও `portfolio_items.album_id` কলাম যোগ করে (আগের flat ক্যাটাগরি-ফিল্টার রিপ্লেস করে)। **উপরের "Portfolio migration"-এর পরে, এই ব্লক রান না করা পর্যন্ত এডমিন প্যানেলে অ্যালবাম তৈরি/আপলোড করা যাবে না**।
- **"Package inclusions migration"** — প্যাকেজে "কী থাকছে" তালিকা যোগ করার জন্য `packages.inclusions` কলাম বানায়। রান করার পর এডমিন প্যানেল → Studio Settings → Package Pricing-এ প্রতিটা প্যাকেজ Edit করে "কী থাকছে" ঘরে এক লাইনে একটা আইটেম লিখে বসাতে পারবেন — কাস্টমার পাবলিক পেজে "প্যাকেজ বেছে নিন" খুললেই এগুলো দেখতে পাবে।

## ডেভেলপার হ্যান্ডওভার

ভবিষ্যতে কোনো ডেভেলপারকে এই প্রজেক্ট বুঝিয়ে দিতে হলে এই সেকশনটাই যথেষ্ট।

**স্ট্যাক (সবই স্ট্যান্ডার্ড, কোনো কাস্টম জিনিস নেই):** React 18 + Vite + Tailwind CSS (ফ্রন্টএন্ড), Supabase (ডাটাবেজ + লগইন + ছবি স্টোরেজ), Vercel (হোস্টিং/ডিপ্লয়), Cloudflare (ডোমেইন DNS)। GitHub-এ `main` ব্রাঞ্চে push হলেই Vercel নিজে নিজে ডিপ্লয় করে।

**ডেভেলপারকে যে ৪টা অ্যাক্সেস দিতে হবে:**
1. **GitHub রিপো** — Settings → Collaborators থেকে invite; পুরো কোড ও ইতিহাস এখানে
2. **Supabase প্রজেক্ট** — Dashboard → Project Settings → Members থেকে invite; ডাটাবেজ, স্টাফ লগইন, পোর্টফোলিওর ছবি সব এখানে
3. **Vercel প্রজেক্ট** — Team Members থেকে invite; ডিপ্লয়মেন্ট, ডোমেইন, env variables এখানে
4. **Cloudflare অ্যাকাউন্ট** — শুধু DNS পরিবর্তনের দরকার হলে

**কোথায় কী আছে:** পাবলিক বুকিং পেজ `src/components/PublicAvailability.jsx`, অ্যাডমিন প্যানেল `AdminPanel.jsx`, পোর্টফোলিও গ্যালারি `PortfolioGallery.jsx` (পাবলিক) + `PortfolioManager.jsx` (অ্যাডমিন), ডাটাবেজ স্কিমা ও সিকিউরিটি রুল `docs/schema.sql` (additive মাইগ্রেশন — পুরনো ব্লক কখনো বদলানো হয় না, নতুন ফিচারে নিচে নতুন ব্লক যোগ হয়), অডিটের ইতিহাস `AUDIT.md`।

## প্রথম স্টাফ লগইন তৈরি

Supabase Dashboard → Authentication → Users → Add user → ইমেইল ও পাসওয়ার্ড দিন (Auto Confirm User অন রাখুন)। প্রতিটা টিম মেম্বারের জন্য আলাদা করে বানাতে পারেন।

## Deploy to Vercel

1. এই GitHub রিপো Vercel-এ Import করুন (Vite অটো-ডিটেক্ট হবে, `vercel.json` লাগবে না)।
2. Vercel Project Settings → Environment Variables-এ `VITE_SUPABASE_URL` আর `VITE_SUPABASE_ANON_KEY` যোগ করুন (`.env.local`-এর মতোই ভ্যালু), তিনটা environment-এই (Production/Preview/Development) টিক দিন।
3. Deploy করুন। পাবলিক লিংক ক্লায়েন্টদের সাথে শেয়ার করুন, `/#/admin` স্টাফদের জন্য।

### "বুকিং লোড করা যায়নি" এরর দেখলে

এর মানে ব্রাউজার Supabase থেকে ডেটা আনতে ব্যর্থ হয়েছে। সাধারণত দুইটার একটা কারণে হয়:
1. উপরের Database setup ধাপ এখনো করা হয়নি (`public_bookings` ভিউ নেই)।
2. Vercel-এ env var সেট করার পর নতুন করে **Redeploy** করা হয়নি — env var যোগ/পরিবর্তন করলে Vercel-এ Deployments ট্যাব থেকে সর্বশেষ ডিপ্লয়মেন্টে গিয়ে "Redeploy" চাপতে হয়, এমনিতে আপডেট হয় না।
