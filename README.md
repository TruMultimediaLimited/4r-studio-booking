# 4R Studio — Booking

স্টুডিও কখন খালি/বুকড তা দেখার একটা পাবলিক পেজ (মাসের ক্যালেন্ডার আকারে), ক্লায়েন্ট নিজে থেকে একটা স্লট বুকিং রিকোয়েস্ট পাঠাতে পারবে, আর টিমের জন্য একটা এডমিন পেজ যেখানে রিকোয়েস্ট কনফার্ম/বাতিল করা যায়।

- পাবলিক পেজ (ক্লায়েন্টদের জন্য): `/`
- টিম পেজ (স্টাফদের জন্য): `/#/admin`

## Local dev

1. `npm install`
2. `.env.example` কপি করে `.env.local` বানান, আপনার Supabase প্রজেক্টের URL ও anon key বসান।
3. `npm run dev`

## Database setup

Supabase Dashboard → SQL Editor-এ গিয়ে `docs/schema.sql` ফাইলের পুরো কনটেন্ট পেস্ট করে "Run" চাপুন। এটা তৈরি করবে:
- `bookings` টেবিল (নাম, ফোন, প্যাকেজ, স্ট্যাটাস — confirmed/pending/cancelled)
- একই সময়ে দুটো বুকিং (বা একটা কনফার্ম্ড + একটা পেন্ডিং রিকোয়েস্ট) ওভারল্যাপ হতে না দেওয়ার ডাটাবেজ-লেভেল গ্যারান্টি
- RLS পলিসি (পাবলিক শুধু ফাঁকা/বুকড দেখতে পারবে ও রিকোয়েস্ট পাঠাতে পারবে, ক্লায়েন্টের নাম/ফোন কখনো পাবলিকলি দেখা যাবে না; শুধু লগইন করা স্টাফ বুকিং কনফার্ম/বাতিল করতে পারবে)
- `public_bookings` ভিউ, যেটা পাবলিক পেজ ব্যবহার করে

এই স্ক্রিপ্টটা যেকোনো সময় আবার রান করা যায় (নতুন ফিচার আসলে, বা টেস্ট ডেটা মুছে ফ্রেশ শুরু করতে চাইলে) — এটা প্রথমে পুরনো টেবিল/ভিউ ডিলিট করে আবার নতুন করে বানায়। **এটা রান করলে তখন পর্যন্ত জমে থাকা সব বুকিং ডেটা মুছে যাবে**, তাই সত্যিকারের বুকিং ডেটা জমার পর অকারণে আর রান করবেন না।

**আগে থেকে চালু ডাটাবেজ থাকলে:** পুরো ফাইল আবার রান করার দরকার নেই — `docs/schema.sql`-এর একদম শেষের **"Security-hardening migration"** ব্লকটা কপি করে একবার রান করলেই হবে (এটা কোনো ডেটা মোছে না)। এই ব্লকটা পাবলিক বুকিং রিকোয়েস্টের সার্ভার-সাইড ভ্যালিডেশন (ফোন/নাম/তারিখ/সময়), স্প্যাম-লিমিট, আর সার্ভার-সাইড দাম হিসাব চালু করে। ফ্রন্টএন্ডের নতুন ভার্সন আর নিজে `total_amount` পাঠায় না — ডাটাবেজই হিসাব করে — তাই **এই ব্লক রান না করা পর্যন্ত নতুন পাবলিক রিকোয়েস্টে দাম/Due রেকর্ড হবে না**।

## প্রথম স্টাফ লগইন তৈরি

Supabase Dashboard → Authentication → Users → Add user → ইমেইল ও পাসওয়ার্ড দিন (Auto Confirm User অন রাখুন)। প্রতিটা টিম মেম্বারের জন্য আলাদা করে বানাতে পারেন।

## Deploy to Vercel

1. এই GitHub রিপো Vercel-এ Import করুন (Vite অটো-ডিটেক্ট হবে, `vercel.json` লাগবে না)।
2. Vercel Project Settings → Environment Variables-এ `VITE_SUPABASE_URL` আর `VITE_SUPABASE_ANON_KEY` যোগ করুন (`.env.local`-এর মতোই ভ্যালু), তিনটা environment-এই (Production/Preview/Development) টিক দিন।
3. Deploy করুন। পাবলিক লিংক ক্লায়েন্টদের সাথে শেয়ার করুন, `/#/admin` স্টাফদের জন্য।

### "বুকিং লোড করা যায়নি" এরর দেখলে

এর মানে ব্রাউজার Supabase থেকে ডেটা আনতে ব্যর্থ হয়েছে। সাধারণত দুইটার একটা কারণে হয়:
1. উপরের Database setup ধাপ এখনো করা হয়নি (`public_bookings` ভিউ নেই)।
2. Vercel-এ env var সেট করার পর নতুন করে **Redeploy** করা হয়নি — env var যোগ/পরিবর্তন করলে Vercel-এ Deployments ট্যাব থেকে সর্বশেষ ডিপ্লয়মেন্টে গিয়ে "Redeploy" চাপতে হয়, এমনিতে আপডেট হয় না।

## এখন কী আছে, পরে কী আসবে

এখন আছে: মাস-ক্যালেন্ডারে স্টুডিও কখন খালি তা দেখা, ওভারল্যাপ হওয়া সম্ভবই না, ক্লায়েন্ট নিজে থেকে একটা স্লটের জন্য রিকোয়েস্ট পাঠাতে পারবে (নাম+ফোন দিয়ে), স্টাফ এডমিন প্যানেল থেকে কনফার্ম/প্রত্যাখ্যান করবে।
পরে যোগ হবে (এই ভার্সনে নেই): প্যাকেজ সিলেকশন থেকে প্রাইসিং দেখানো, অ্যাডভান্স পেমেন্ট (২৫%) নেওয়ার ব্যবস্থা।
