## 🔴 জরুরি / গুরুত্বপূর্ণ (নতুন ফাইন্ডিং)

### N1. এডমিনের ডেটা *লোডিং*-এ এরর নীরবে হারায়
`AdminPanel.jsx:532-558` — `loadBookings`, `loadPayments`, `loadStatusLogs`, `loadPackages`, `loadSettings`, `loadOffDays` সবগুলো `const { data } = ...` করে `error` উপেক্ষা করে। নেটওয়ার্ক/RLS সমস্যায় এডমিন দেখবে খালি লিস্ট ("No bookings") — বুঝবে না ডেটা আনতেই ব্যর্থ হয়েছে। রাউন্ড ১-এ অ্যাকশনগুলো ঠিক হয়েছিল, লোডগুলো বাদ পড়েছিল। `PublicAvailability.jsx`-এর `loadConfig`-ও একই রকম (তবে সেখানে ডিফল্ট-ফলব্যাক আছে বলে কম ক্ষতিকর)।
**করণীয়:** লোড ব্যর্থ হলে `actionError` ব্যানারে দেখানো + Retry বাটন।

### N2. ডিপেন্ডেন্সি দুর্বলতা: `vite` (high), `esbuild` (moderate)
`npm audit`: vite ≤6.4.2-এ path-traversal (high) ও esbuild dev-সার্ভার ইস্যু (moderate)। **দুটোই শুধু `npm run dev`-এর লোকাল dev সার্ভারে প্রযোজ্য** — Vercel-এর প্রোডাকশন স্ট্যাটিক সাইটে exploit করার পথ নেই, তাই এখনই বিপদ নেই। তবে পরিকল্পিতভাবে vite 7/8 + @vitejs/plugin-react 6-এ আপগ্রেড করা উচিত (breaking change, আলাদা করে টেস্টসহ)।

## 🟠 মাঝারি (রাউন্ড ১ থেকে সচেতনভাবে স্থগিত + নতুন)

- **টাইপোগ্রাফি/অ্যাক্সেসিবিলিটি:** ৯–১০px টেক্সট এখনো ৩৮ জায়গায়; `text-[#333333]/45`-জাতীয় নিম্ন-কনট্রাস্ট টেক্সট; ক্যালেন্ডার/এডমিন বাটনের টাচ টার্গেট ~২৪-২৮px (সুপারিশ ৪৪px)।
- **এডমিন পেজিনেশন নেই:** লগইনেই `bookings`+`payments`+`booking_status_log`-এর *সব* রো লোড হয়। ৬-১২ মাসের ডেটা জমলে ধীর হবে। তারিখ-রেঞ্জ ফিল্টার/লিমিট দরকার হবে।
- **`get_bookings_by_phone` enumeration:** ফোন নম্বর জানলে যে কারো বুকিং+প্যাকেজ (দামসহ) দেখা যায়, রেট-লিমিট নেই। ছোট ব্যবসার মেনে নেওয়া trade-off; চাইলে ভবিষ্যতে OTP।
- **`booking_status_log.changed_by` ক্লায়েন্ট-সাপ্লায়েড:** স্টাফ চাইলে অন্যের ইমেইল বসাতে পারে; DB trigger-এ `auth.email()` নিলে অডিট ট্রেইল শক্ত হয়।
- **`package_name`-এ ক্লায়েন্ট-গণনা দামের টেক্সট:** সার্ভার এখন `total_amount` নিজে হিসাব করে, কিন্তু `package_name` স্ট্রিংয়ের ভেতরের "X hr = Y Tk" অংশটা এখনো ক্লায়েন্টের লেখা — tamper করলে টেক্সটে ভুল দাম দেখাতে পারে (হিসাবের ক্ষতি নেই, শুধু বিভ্রান্তি)। সার্ভারে শুধু লেবেল রাখা ভালো।
- **এডমিনের Add Booking অফ-ডে সতর্কতা দেয় না** (স্টাফের জন্য ইচ্ছাকৃত অনুমোদিত, তবে একটা সতর্কবার্তা ভালো)।
- **ভাষা:** পাবলিক UI পুরোটা ইংরেজি — বাংলাদেশি কাস্টমারের জন্য মূল ফ্লো বাংলায়/দ্বিভাষিক করলে কনভার্শন বাড়বে। (Hind Siliguri ফন্ট লোড হচ্ছেই।)
- **JSON-LD `openingHours` হার্ডকোডেড** `Mo-Su 09:00-23:00` — এডমিন সেটিংসে সময় বদলালে এটা পুরনোই থেকে যাবে (`index.html:48`)।
- **গ্লোবাল ২০/ঘণ্টা pending-লিমিট:** আক্রমণের সময় বৈধ কাস্টমারও সাময়িক আটকাতে পারে — জানা trade-off, নজরে রাখুন।

## 🟡 ছোট / পরে করলেও চলে

- **og:image = logo.png** — শেয়ার প্রিভিউতে ছোট লোগো দেখায়; ১২০০×৬৩০ ব্যানার (স্টুডিওর ছবি+লোগো+"Book Online") বানালে Facebook/WhatsApp-এ অনেক আকর্ষণীয় দেখাবে, `twitter:card`-ও `summary_large_image` করা যাবে।

- **sitemap `lastmod` হার্ডকোডেড** (2026-07-17) — বাসি হতে থাকবে; হয় ফিল্ডটা বাদ দিন নয় বিল্ডে জেনারেট করুন।

- **ডেস্কটপ লেআউট** এক সরু কলাম — বড় স্ক্রিনে ক্যালেন্ডার+ফর্ম পাশাপাশি দিলে পরিণত দেখাবে।
- **মেজর ভার্সন পিছিয়ে:** React 18→19, Tailwind 3→4, Vite 5→8 — এখন সমস্যা নেই, বছরে একবার পরিকল্পিত আপগ্রেড ভালো।
